# Skill: /test-plan — 测试方案自动生成

## 功能

基于风险评估结果（RPN≥27 的风险项），自动从知识库检索匹配的测试项目，填充参数/样品数/标准，生成结构化测试方案。如用户提供已排方案，自动比对并输出差异分析。

## 输入

- `/risk-eval` 的输出（风险列表，含 S/O/D/RPN + 关联测试）
- 原始器件信息（名称、材料、工艺、规格书）
- **用户已排测试方案（可选）**：如用户输入文件中包含已排好的测试项目列表

## 执行流程

### Step 1: 筛选需测试的风险

取 RPN ≥ 27（中 及以上）的全部风险项。

### Step 1.5: 用户方案提取（如用户提供）

如果用户输入中包含已排好的测试方案，提取以下字段：
- 测试名称
- 测试参数/条件
- 样品数
- 接收标准
- 备注/说明

暂存为 `user_test_plan`，用于后续比对。

### Step 2: 检索测试项目

对每个风险，从以下知识库检索匹配的测试项：
1. `memory/failure-database/failure-modes-database.md` 的"拦截测试"字段 → 直接对应
2. `memory/failure-database/testing-sop.md` 的测试项目 → 关键词匹配
3. `memory/test-items.md` 的 `applicable_failure_modes` → 失效模式名称匹配

### Step 3: 填充测试参数

| 字段 | 来源优先级 |
|------|------|
| test_name | testing-sop.md > test-items.md > 工程推定 |
| priority | 致命缺陷(S)/高风险(A)→P0, 中风险(B)→P1, 低风险(C)→P2 |
| parameters | **①规格书要求** > ②testing-sop.md 默认参数 > ③同品类历史测试参数 > ④工程推定 |
| sample_size | 致命缺陷(S)≥10pcs, 高风险(A)≥5pcs, 中风险(B)≥3pcs, 低风险(C)≥1pcs（或引用 testing-sop.md 默认值，取较大者） |
| acceptance_criteria | **①规格书标准** > ②testing-sop.md 接收标准 > ③行业/国家标准 > ④"建议参照竞品标准或与终端协商确认" |
| derived_from_failure_modes | 该测试覆盖的失效模式列表 |
| standard_ref | GB/T / ISO / ASTM / IEC 编号 |

### Step 4: 去重与合并

- 同一测试项覆盖多个风险 → 保留一条，取最高优先级
- 多个测试项可合并为同一试验 → 合并并注明覆盖的风险

### Step 5: 排序

按优先级排列：高 → 中 → 低。同优先级按 RPN 降序。

### Step 6: 用户方案 vs AI方案比对（如用户提供）

逐项比对，按以下场景分类：

| 比对场景 | 差异类型 | 处理方式 |
|---------|---------|---------|
| 用户有 AND AI有，名称一致 | 一致 / 参数差异 | 参数一致→"一致"；参数不同→"参数差异"+注明差异 |
| 用户有 AND AI有，名称不同但覆盖同一风险 | 等效替换 | 注明AI对应测试项+判断依据 |
| 用户有 BUT AI无 | 用户独有 | AI评估RPN<27无需测试，或知识库无匹配测试项 |
| 用户无 BUT AI有 | AI补充 | RPN≥27且用户未覆盖，标注"【建议补充】" |

**判断依据要求**：
- 每条差异必须附带一句话依据，说明为何存在该差异
- 依据来源优先级：failure-modes-database.md > testing-sop.md > 工程通用原则
- 用户遗漏 关键/高 风险的必做测试 → 强制标注"【建议补充】"并说明遗漏的风险项

## 输出格式

```markdown
## 测试方案 — <器件名>

| 优先级 | 测试名称 | 测试参数 | 样品数 | 接收标准 | 关联失效模式 | 标准参考 |
|--------|---------|---------|--------|---------|-------------|---------| 
| 关键 | 百格附着力 | 百格刀(1mm), 3M 600#胶带 | 5pcs | ≥4B (≤5%脱落) | FM-001 | GB/T 9286 |
| 中 | 表面能试验 | Arcotest达因笔34 | 3pcs | 5s内不收缩 | FM-004 | - |

### 用户方案 vs AI方案比对

| 序号 | 比对项 | 用户方案 | AI方案 | 差异类型 | 判断依据 |
|------|--------|---------|--------|---------|--------|
| 1 | 百格附着力 | 3pcs, 48h | 5pcs, 96h | 参数差异 | 用户样品数偏少，关键风险≥5pcs；testing-sop.md规定96h |
| 2 | UV光照 | 无 | 有, UVA-340 | 【建议补充】 | FM-UV光照变色RPN=125(关键)，用户方案未覆盖 |
| 3 | 盐雾试验 | 有, 96h | 有, 96h | 一致 | 双方参数和标准一致 |

### 样品总量
- 总计: XX pcs（含对照组）
- 建议样品来源: 试产批 / 同一模号

### 测试顺序建议
1. 非破坏性（外观 → 尺寸）→ 2. 环境测试（盐雾/恒温恒湿/温度冲击）→ 3. 破坏性（百格/耐磨/跌落/挤压）
```

## 处理规则

- 致命缺陷(S)/高风险(A) 风险必须有对应的测试手段覆盖，无匹配测试项时标注"【待补充】— 需人工设计"
- 测试参数严格按Step 3来源优先级填充，优先使用规格书要求
- 接收标准不能为空，无明确标准时按优先级链填充
- **异常处理**：用户方案格式异常或无法解析时，标注"用户方案解析失败，已跳过比对"，继续执行AI方案生成
- 样本量不足时标注"样品量偏少，建议增加"
- **用户方案比对为强制步骤**：如用户提供已排方案，必须执行Step 6，差异项必须写判断依据
- **安全优先**：用户遗漏 致命缺陷(S)/高风险(A) 风险的必做测试 → 强制标注"【建议补充】"并说明遗漏的风险项
