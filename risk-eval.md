# Skill: /risk-eval — 风险自动评估

## 功能

基于器件品类、材料、工艺信息，从知识库检索失效模式，进行匹配适用性判断，对每个适用失效模式打出 S/O/D 分数并计算 RPN，最后通过规则引擎交叉验证和安全红线检查，输出完整风险评估表。

## 输入

- `/classify` 的输出（品类）
- `/checklist` 的输出（Checklist 评估结果）
- 原始器件信息（名称、材料、工艺、规格书内容）

## 传音企业 RPN 评分标准（官方标准，强制使用）

### 风险等级阈值

计算方法：R.P.N = S × O × D

| 等级 | 符号 | 阈值 | 对应颜色 |
|------|------|------|----------|
| 致命缺陷 | S | RPN ≥ 125 | 红色 |
| 高风险 | A | 75 < RPN < 125 | 红色 |
| 中风险 | B | 27 < RPN ≤ 75 | 黄色 |
| 低风险 | C | RPN ≤ 27 | 绿色 |

### Severity 严重度（1-5分制）

| 级别 | 标准 | 后果 |
|------|------|------|
| 5 | 方案存在重大缺陷，没有解决方案和改善计划；无法进行生产或测试； | 致命问题，限制用户使用或无法正常使用，用户无法接受，可能导致市场召回的问题；生产或测试无法继续。 |
| 4 | 方案存在严重缺陷，部分功能受到影响，测试无法进行；有解决方案但会影响进度或引起其它故障，导致解决方案无法导入； | 严重问题，严重影响用户使用体验，用户可能会退货及投诉； |
| 3 | 方案存在缺陷，测试或开发受到部分影响但可以继续进行。 | 一般问题，用户没有受到明显影响，可能会产生抱怨及负面影响； |
| 2 | 外观类问题，非功能性问题；对设计开发或测试没有直接影响，且可以快速改善。 | 对用户来说有轻微的感知，改善后可提升用户体验； |
| 1 | 问题现象及影像用户无法感知 | 无 |

### Occurrence 频度（1-5分制）

| 级别 | 标准 | 概率 |
|------|------|------|
| 5 | 发生频率极高，历史数据/量产案例证实 | > 20% | 极高 |
| 4 | 发生率较高，有已知案例或相似组合失效记录 | > 1%, < 20% | 高 |
| 3 | 偶发失效，少数案例或工程推定 | > 1/500, < 1% | 中 |
| 2 | 低频失效，成熟工艺中偶见 | > 1/2000, < 1/500 | 低 |
| 1 | 极低频，长期量产验证通过 | < 1/2000 | 极低 |

### Detection 探测度（1-5分制）

| 级别 | 标准 | 探测能力 |
|------|------|----------|
| 5 | 没有明确的测试方案或缺少测试设备进行检测；现有测试模块（集成/硬件/软件/可靠性/批量生产）基本无法完成对不良品和分析进行检测和拦截。 | 完全不确定 |
| 4 | 测试方案完成基本制定，缺少详细测试用例，相应设备及环境已经制定采购计划；现有测试模块（集成/硬件/软件/可靠性/批量生产）对不良品和分析进行检测和拦截的可能性很小。 | 低 |
| 3 | 测试方案准备完成，相应设备及环境搭建中，测试人员培训未进行；现有测试模块（集成/硬件/软件/可靠性/批量生产）对不良品和分析进行检测和拦截的可能性较高。 | 中 |
| 2 | 测试方案、设备、人员全部准备完成；现有测试模块（集成/硬件/软件/可靠性/批量生产）相关用例可以覆盖或检测故障或风险； | 高 |
| 1 | 测试方案、设备、人员全部准备完成；现有测试模块（集成/硬件/软件/可靠性/批量生产）可以正常检测故障或风险。 | 几乎确定 |

### 重要说明

1. **传音企业使用 1-5 分制，不是 1-10 分制**。所有 S/O/D 评分必须按上述 1-5 分标准执行。
2. **风险等级命名**：致命缺陷(S) / 高风险(A) / 中风险(B) / 低风险(C)，报告输出中使用此命名。
3. **RPN 计算**：R.P.N = S × O × D，阈值与上述表一致。

## 执行流程

### Step A: 加载失效模式

读取 `memory/failure-database/failure-modes-database.md`，按品类、物料中类、物料小类筛选适用的失效模式。

如果知识库中无该品类记录，基于工程通用原则推定并在输出中标注"知识库无该品类记录（推定）"。

**重要：用户输入源中已明确识别的风险必须纳入评估。**
- 如果 PPTX/规格书中明确写了某项风险（如"BTB无固定，跌落可靠性无法实现"），必须将其作为失效模式纳入评估，并给予高S/O评分。
- 不要忽略用户在输入文件中自己已经识别的风险，这是最重要的证据源。
- 对多组件复合体（如三合一组件），要评估每个组件的独立失效模式 + 组件间界面的复合失效模式。

### Step B: 失效模式匹配

对每个失效模式，判断是否适用于当前器件：

| 匹配条件 | 匹配类型 | 说明 |
|---------|---------|------|
| 器件材料 在失效模式的"易受影响材料"中 | 直接匹配 | 材料关键词命中 |
| 器件工艺 在失效模式的"易受影响工艺"中 | 直接匹配 | 工艺关键词命中 |
| Checklist 中有一项以上关联到此失效模式 | 间接匹配 | 增强置信度 |
| 历史记录中有相似器件发生此失效 | 间接匹配 | 历史案例支持 |
| 通用适用（该品类几乎所有器件都需评估） | 默认适用 | 如"外观件→外观失效" |

匹配结果标注：
- **适用** — 至少一个直接匹配条件满足
- **潜在适用** — 仅有间接匹配
- **不适用** — 无任何匹配（需备注理由，不纳入评分）

### Step C: S/O/D 量化评分

**强制使用传音企业官方标准（1-5 分制），见上方「传音企业 RPN 评分标准」章节。**

简要引用：

- **S (1-5)**: 5=重大缺陷/无法生产测试 → 4=严重缺陷/影响功能 → 3=有缺陷但可继续 → 2=外观类/非功能/可快速改善 → 1=用户无法感知
- **O (1-5)**: 5=>20%极高 → 4=>1%~20%高 → 3=>1/500~1%中 → 2=>1/2000~1/500低 → 1=<1/2000极低
- **D (1-5)**: 5=无测试方案/完全不确定 → 4=方案基本制定/低 → 3=方案完成/中 → 2=方案设备人员就绪/高 → 1=全部就绪/几乎确定

评分依据要求：每个分数必须附带一句话依据，不得使用"基于经验"等模糊表述。引用知识库条目时注明来源。推定的分数末尾标注"（推定）"。

### Step D: RPN 计算与定级

```
RPN = S × O × D（传音 1-5 分制）

致命缺陷(S) : RPN ≥ 125   → 红色
高风险(A)   : 75 < RPN < 125 → 红色
中风险(B)   : 27 < RPN ≤ 75  → 黄色
低风险(C)   : RPN ≤ 27      → 绿色
```

### Step E: 规则引擎交叉验证

1. 读取 `memory/rules.md`
2. 检查所有规则的 condition 是否与当前器件属性（品类/材料/工艺/RPN等级/失效模式）匹配
3. 匹配的规则 → 将其结论合并到风险列表（如 S-score 调整、必做测试增加）
4. 交叉比对：
   - Agent评估 有 AND 规则有 → 确认，置信度增强
   - Agent评估 有 BUT 规则无 → 标注"AI推定风险"
   - Agent评估 无 BUT 规则有 → 重新审视是否遗漏，补充评估

### Step F: 安全红线检查

按 SOUL 定义的安全红线逐条检查，触发红线的风险在输出中用 **【安全红线】** 标记：

| 红线 | 触发条件 |
|------|---------|
| 人身安全风险 | 失效模式涉及触电/起火/爆炸/化学泄漏/机械伤人 |
| 认证阻断风险 | 可能导致 CCC/UL/CE/SRRC/PSE 等强制认证失败 |
| 历史重复失效 | 同品类器件在过去评审中 ≥3 次出现相同失效模式 |
| 安规器件核心项 | 电池类 高风险(A)及以上 且涉及过充/短路/热失控 |
| 无测试覆盖 | 某项高风险(A)及以上（RPN>75）无对应测试手段可检出 |

```markdown
| 1 | 【安全红线】FM-001 涂层附着力不足 | 直接匹配(材料+工艺) | 5 | 方案存在重大缺陷，无法进行生产测试 | 4 | 历史不良率16.7% (>1%) | 3 | 测试方案准备中，拦截可能性较高 | 60 | 中风险(B) | Agent+规则 |

### 安全红线警告
| 红线 | 触发项 | 详情 |
|------|--------|------|

```
