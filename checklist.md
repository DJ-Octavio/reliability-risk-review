# Skill: /checklist — 自动 Checklist 评估

## 功能

加载品类对应的 Checklist 模板，或基于知识库失效模式自主生成检查项，逐项自动评估，无需人工确认。

## 输入

从上一个 Skill (`/classify`) 的输出 + 用户原始输入获取：
- `category`: 品类（如"结构件"）
- 器件名称、材料、工艺
- 规格书内容（如用户提供了文件）

## 执行流程

### Step 1: 加载与生成检查项

1. 尝试读取 `memory/checklist-templates/<品类>-checklist.md`（如存在则优先使用）
2. 尝试读取 `memory/failure-database/failure-modes-database.md`，按品类筛选适用的失效模式
3. 如果两者都不存在 → 按下方「自主生成检查项标准」生成

**重要：从用户输入（PPTX/规格书/图纸）中提取器件组成信息。**

**多组件复合体规则**：识别到器件为多组件复合体（名称含"三合一"/"+"/材料跨品类/工艺涉及贴合）时，必须额外生成以下维度：
- 组件间贴合/附着界面可靠性
- 新增连接器（BTB/FPC）的固定与可靠性
- 新增电气元器件的ESD/寿命/热设计
- 供应商协同风险（首次多方协同）

### Step 2: 检查项维度

检查项按以下维度自动分类（至少覆盖以下6个维度）：

| 维度 | 说明 |
|------|------|
| 器件测试履历 | 上代客诉/EVT-DVT测试历史/相似机型失效 |
| 材料与工艺 | 材料选型合理性、工艺匹配度、关键工艺控制点 |
| 表面处理 | 涂镀层/丝印/镭雕/转印的附着力/耐磨性/耐蚀性 |
| 结构强度 | 弯折/跌落/冲击/孔位/薄弱结构的强度 |
| 环境可靠性 | 恒温恒湿/温度冲击/盐雾/UV光照后的性能保持 |
| 装配匹配 | 间隙段差/配合公差/辅料贴附/连接器可靠性 |

### Step 3: 逐项评估

对每一项，基于以下信息源（按优先级）自主判断：

| 优先级 | 信息源 | 操作 |
|--------|--------|------|
| P0 | 规格书中的技术参数或设计说明 | 直接匹配检查项 |
| P1 | `memory/failure-database/failure-modes-database.md` | 查询该材料/工艺/区域的已知失效模式 |
| P2 | `memory/failure-database/structural-process-table.md` | 查询该材料×工艺组合的关键控制点 |
| P3 | `memory/rules.md`（如存在） | 检查是否有规则条件匹配当前器件 |
| P4 | `memory/history/<品类>/`（如存在） | 查询相似器件的历史高频检查项 |
| P5 | 工程通用原则 | 基于该品类器件的通用工程经验 |

### Step 4: 输出结果

每项输出 `{AI判断, 依据, 置信度}`

- **AI判断**: ✅ 建议关注 / ❌ 无需关注 / N/A
- **依据**: 一句话说明，引用知识库文件+条目名
- **置信度**: 确认（知识库明确匹配）/ 推定（同品类普遍规律）/ 待确认（信息不足）

### 自主生成检查项标准（知识库缺失时）

当品类无对应checklist模板且失效数据库无记录时，按以下默认清单生成：

| 维度 | 默认检查项（至少3项/维度） |
|------|--------------------------|
| 器件测试履历 | ①上代机型同位置是否有客诉/失效记录 ②相似品类历史评审是否出现相同失效 ③当前是否有EVT/DVT测试历史 |
| 材料与工艺 | ①材料×工艺组合是否有已知失效风险 ②关键工艺控制点是否可检出 ③供应商是否具备该材料/工艺量产经验 |
| 表面处理 | ①涂镀层附着力是否达标（百格/拉拔） ②耐磨性是否满足使用寿命 ③耐蚀性是否通过盐雾验证 |
| 结构强度 | ①跌落强度是否满足标准（1.5m六面四角） ②弯折/冲击薄弱点是否识别 ③孔位/卡扣结构强度是否验证 |
| 环境可靠性 | ①恒温恒湿后功能是否保持 ②温度冲击后是否开裂/分层 ③盐雾后是否腐蚀/失效 |
| 装配匹配 | ①间隙段差是否符合外观标准 ②辅料贴附位置是否正确 ③连接器/接口可靠性是否验证 |

### Fallback 路径（按优先级）

1. `memory/checklist-templates/<品类>-checklist.md` → 直接使用模板
2. `memory/failure-database/failure-modes-database.md` → 从失效模式反推检查项
3. `memory/history/<品类>/` → 从历史评审记录提取高频检查项
4. 上方「自主生成检查项标准」→ 按默认清单生成，所有条目标注"推定"

**知识库全部缺失时**：按Fallback路径4生成，每条检查项标注"置信度：推定（知识库无记录）"。

## 输出格式

```markdown
## Checklist 评估结果 — <器件名>

### 器件测试履历
| # | 检查项 | AI判断 | 依据 | 置信度 |
|---|--------|--------|------|--------|
| 1 | 上代机型同位置是否有客诉/失效记录 | ✅ 建议关注 | 未提供上代机型信息，默认建议追溯 | 待确认 |

### 材料与工艺
| # | 检查项 | AI判断 | 依据 | 置信度 |
|---|--------|--------|------|--------|
| 2 | 复合板层间结合力是否达标 | ✅ 建议关注 | failure-modes-database.md: 转印层弯折裂纹风险 | 确认 |
```

## 处理规则

- **不跳项**：每项都必须评估，即使不适用也要给出 N/A 理由
- **不合并**：每个检查项独立判断
- **关联追踪**：关联失效模式列出的 FM-XXX，必须在风险评估阶段（/risk-eval）逐一匹配
- **置信度标注规则**（来自 SOUL 第9条）：
  - **确认** — 知识库有明确匹配 + 工程推理支持
  - **推定** — 基于同品类/同材料的普遍规律，但未找到直接证据
  - **待确认** — 信息不足，需要在后续步骤中关注
- **规格书优先**：如果用户提供了规格书，规格书中的技术参数直接作为判断依据
- **知识空白标注**：知识库中无对应品类记录时，明确标注"知识库无记录（推定）"
- **多组件复合体**：识别到多组件时，必须额外覆盖组件间界面、新增连接器、ESD/热设计、供应商协同等维度，不得遗漏
