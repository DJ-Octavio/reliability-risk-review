# Skill: /report — xlsx 报告生成

## 功能

汇总所有前置 Skill 的输出，生成标准化 xlsx 格式报告，包含 6 个 Sheet + 可选的测试方案比对 Sheet。

## 输入

- 所有前置 Skill 的输出：
  - `/classify` → 品类（结构件/辅料/电声器件/光学器件/包材/安规器件）
  - `/checklist` → Checklist 评估（每项 AI判断/依据/置信度）
  - `/risk-eval` → 风险评估（失效模式/S/O/D/RPN/等级/安全红线）
  - `/test-plan` → 测试方案（优先级/参数/样品数/标准/用户方案比对）

## 执行流程

### Step 1: 汇总输入

收集所有前置输出，检查数据完整性。

**强制：从用户输入源（PPTX/规格书/图纸/消息文本）中主动提取器件基本信息，不得留空或写"未提供"。**

当用户提供 PPTX 可行性评估报告/规格书/图纸/消息文本时，必须从中提取以下信息填入 Sheet 1：

| 字段 | 提取来源优先级 |
|------|---------------|
| 器件名称 | 文件名/PPT标题/用户消息 → 提取 |
| 器件组成 | PPT中的叠层图/BOM清单/系统框架图 → 提取具体组件清单 |
| 项目名称 | PPT标题/正文 → 提取完整项目名 |
| 预研机型/平台 | PPT正文 → 提取（如X6878/NOTE 60 Pro） |
| 材料体系 | PPT中的材料描述/规格书 → 逐项列出所有材料（PC/PMMA/PET/OCA/FPC/泡棉等） |
| 工艺链路 | PPT中的工艺流程/制造维度 → 列出完整工艺顺序 |
| 供应商清单 | PPT采购维度 → 逐个列出供应商名称和对应职责 |
| 关键器件型号 | PPT硬件维度 → 提取IC型号、sensor型号等 |
| 结构参数 | PPT叠层图/结构维度 → 提取厚度、像素数、位置信息等 |
| 电气参数 | PPT安规/功耗页 → 提取电压、电流、功耗等 |

如果某字段确实无法从任何输入源获取，才标注"未提供"。**禁止在已有PPTX等详细输入时仍大量留"未提供"。**

### Step 2: 生成 6-Sheet 报告

**严格按上方「📋 标准模板（2026-06-01 用户确认锁定版）」中定义的 Sheet 结构、列名、编号格式、着色规则执行。以下是补充说明：**

#### Sheet 1: 基本信息

按锁定模板定义。除用户输入的基础字段外，还应尽可能从 PPTX/规格书中提取：器件本质、结构叠层、新增模具、关键阻断项等行。

#### Sheet 2: Checklist

按锁定模板定义。编号格式：C1, C2, C3... ✅行绿底(#A5D6A7)，❌行红底(#FF6B6B)。

#### Sheet 3: 风险评估

按锁定模板定义。S/O/D 使用传音1-5分制。**每个 S/O/D 分值必须附带一句话依据**（S依据/O依据/D依据三列），RPN 必须降序排列。等级着色：致命缺陷(S)=#FF6B6B, 高风险(A)=#FFA500, 中风险(B)=#FFD700, 低风险(C)=#90EE90。安全红线行在失效模式前加"【安全红线】"。

#### Sheet 4: 差异分析

按锁定模板定义。编号格式：D1, D2, D3... 置信度着色：确认=#A5D6A7，推定=#FFD700，待确认=#FFA500。

#### Sheet 5: 测试方案

按锁定模板定义。编号格式：T1, T2, T3... 优先级着色：P0=#FF6B6B, P1=#FFA500, P2=#FFD700, P3=#90EE90。

**如用户提供已排测试方案**，追加 Sheet 5B: 测试方案比对（见上方锁定模板定义）。

#### Sheet 6: 总体意见

按锁定模板定义。编号格式：F1, F2, F3... 关键发现必须带emoji分级（🔴/🟠/🟡）。必须包含：关键发现列表 + 高风险数量统计 + 明确结论三选一 + 后续动作清单。结论行深灰底+粗体。

### Step 3: xlsx 文件生成

使用 `openpyxl` 库生成带格式的多 Sheet xlsx 文件。

**步骤**：
1. 创建 workbook，按锁定模板定义创建 6 个 Sheet（使用 `openpyxl.Workbook()` + `wb.create_sheet()`）
2. 写入表头（深蓝底 RGB:68,114,196 + 白字 + 粗体）和数据行
3. 按锁定模板的着色规则设置行底色（风险等级/判断/置信度）
4. 设置列宽（自适应内容，最小12字符）、冻结首行（`ws.freeze_panes = 'A2'`）、细边框
5. 保存到 `reports/<器件名>_<项目名>_<YYYYMMDD>.xlsx`
6. 通过 `feishu_drive_file(action=upload)` 上传到飞书云空间
7. 将飞书云空间的 xlsx 文件 URL 通过飞书消息发送给用户

**关键格式设置**（openpyxl）：
- 表头：`Font(name='Arial', bold=True, color='FFFFFF')` + `PatternFill(start_color='4472C4')` + `Alignment(wrap_text=True)`
- 数据行：`Font(name='Arial', size=10)` + `Alignment(wrap_text=True)` + `Border(all sides thin)`
- 风险等级着色：致命缺陷(S)=#FF6B6B, 高风险(A)=#FFA500, 中风险(B)=#FFD700, 低风险(C)=#90EE90
- Checklist：✅行绿底(#A5D6A7)，❌行红底(#FF6B6B)
- 置信度：确认=#A5D6A7, 推定=#FFD700, 待确认=#FFA500

## 兜底规则

当所有知识库（失效模式/checklist/测试项/规则）均缺失时，按以下兜底策略生成最小可用报告：
- Sheet 1：从用户输入源提取，不依赖知识库
- Sheet 2-5：基于工程通用原则生成最小检查项清单（每个维度至少3项），所有条目标注"知识库无记录（推定）"
- Sheet 6：结论标注"知识库不完整，以下推定结论需人工验证"
- **不得因知识库缺失而中断报告生成**

## 处理规则

- 报告必须包含全部 6 个 Sheet（或 7 个含测试方案比对表），Sheet 6「总体意见」强制不可跳过
- 各 Sheet 的列名、列顺序必须与锁定模板完全一致
- S/O/D 使用传音1-5分制，RPN 降序排列
- 安全红线行标注"【安全红线】"
- 总体结论必须三选一（建议通过/建议有条件通过/不建议通过）
- xlsx 文件通过 `feishu_drive_file(action=upload)` 上传到飞书云空间，将文件 URL 链接发送给用户（不发文件附件）

## 历史经验教训

### 2026-05-30 — 基本信息缺失导致用户强烈不满

**问题**：用户发来一份 45 页的 PPTX 可行性评估报告（敲击流光共鸣项目），里面包含器件组成、材料、工艺、供应商、IC型号、结构厚度等大量信息。第一次评审时没有从 PPT 中提取这些信息，Sheet 1 写了大量"未提供"，导致用户直接反馈"器件基本信息都没有，分析的也不好"。

**根本原因**：
1. 只读了 PPT 的 markitdown 文本提取，但深度不够（很多关键信息在图片/表格中）
2. 没有从 PPT 中提取器件基本信息，而是假设信息不足
3. 评审对象识别错误 — 这不是一个普通电池盖，而是"电池盖+TP+miniLED灯带/点阵"三合一组件

**修复方案**：
1. 使用 `zipfile + xml` 方式深度解析 PPTX 全部 slide 文本内容
2. 从 PPT 中提取所有器件相关参数（项目名、机型、材料、工艺、供应商、IC、结构参数）
3. 正确识别器件本质（三合一组件 vs 单体电池盖）

**教训（必须记住）**：
- **用户提供 PPTX = 器件信息源，不是信息不足**。要从 PPT 中主动提取，而不是说"未提供"。
- **器件组成要搞清楚** — 电池盖 ≠ 单体电池盖，要识别出实际组成（本体+TP+LED+FPC+BTB等）。
- **供应商信息要从采购维度页面提取** — PPT 通常有专门的采购/制造维度页面列出供应商。
- **IC/sensor/关键器件型号要从硬件维度页面提取**。
- **结构厚度/BOM/像素数等要从叠层图页面提取**。
